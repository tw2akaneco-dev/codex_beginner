# 大类资产报告 HTML 展示规范

生成 Markdown 报告后，必须同时生成同名 HTML 文件，保存在同一目录下。例如：
- `YYYY-MM-DD_早间大类资产日报.md`
- `YYYY-MM-DD_早间大类资产日报.html`

HTML 需要是可直接用浏览器打开的完整页面，不依赖外部 CDN、远程字体或前端构建工具。请把 CSS 写在 `<style>` 中，把正文写成语义化 HTML。

如需从 Markdown 自动渲染 HTML，优先使用本地脚本：

```bash
node /Users/px/Desktop/codex_beginner/日报/render_report_html.js 输入.md
```

脚本会生成同名 `.html`，并使用本规范的左侧目录布局。

## 页面气质

- 排版温和、清晰、克制，不要像卖方研报，也不要像营销页。
- 使用浅色正文背景、白色内容区、低饱和边框和柔和强调色。
- 左侧目录栏可以使用深墨绿色或深蓝绿色，参考“课程讲义 / 学习型看盘页面”的质感。
- 适合学生学习宏观和大类资产：信息层次清楚，读起来有“先总后分、先结论后证据”的逻辑。
- 不使用夸张渐变、大面积深色、装饰性背景图、闪烁动画。

## 结构要求

- 桌面端必须使用左右双栏布局：
  - 左侧为固定目录栏，宽度建议 280-320px，深色背景，满高展示。
  - 右侧为正文内容区，浅色背景，最大阅读宽度建议 1160-1280px。
  - 左侧目录栏必须包含报告标题、日期/类型标签、目录列表。
  - 目录项必须是可点击的锚点链接，点击后跳转到右侧对应章节。
- 移动端必须改为单栏布局：
  - 左侧目录栏变成顶部信息区，不固定在屏幕左侧。
  - 目录列表可以横向或纵向展示，但仍然保留锚点跳转。
- 顶部或左侧包含报告标题、日期、报告类型和一句简短摘要。
- 提供目录导航，锚点链接到各章节；目录顺序必须与正文 h2 章节一致。
- 章节之间要有明显间距，每节使用轻卡片或分隔线，不要厚重，圆角不超过 8px。
- 每节末尾的“小白理解”使用浅色提示框展示。
- 表格放在可横向滚动容器中，避免移动端溢出。
- 对“异常波动”“风险”“待确认”“数据缺失”使用温和但清晰的标签样式。
- 正文结构要突出逻辑：先总览，再数据，再新闻，再联动，再观察清单。

## CSS 建议

- 字体使用系统字体：`-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`。
- 正文字号 15-16px，行高 1.65-1.8。
- 页面整体使用 `display: grid; grid-template-columns: 300px minmax(0, 1fr);` 或等价布局。
- 左侧栏使用 `position: sticky; top: 0; height: 100vh; overflow-y: auto;`，方便长报告阅读。
- 正文区域使用 `max-width: 1180px; margin: 0 auto; padding: 40px 48px;`。
- 主色可以使用低饱和蓝灰或青色；风险提示可用柔和红/橙。
- 表格表头固定浅灰背景，单元格边框细线。
- 小标题可以用左侧短色条或细分割线增强层次，不要使用大面积色块。
- 移动端适配：小屏下使用单列，减少边距，目录改为顶部单列，表格保持横向滚动。

## 左侧目录栏要求

- 左侧目录栏背景建议：`#0f2f2d`、`#103735` 或类似深墨绿色。
- 左侧标题用 28-34px，颜色接近白色。
- 左侧摘要用 14-16px，颜色使用半透明浅色。
- 日期、报告类型、数据状态可以做成圆角胶囊标签。
- 目录标题写“目录”。
- 目录链接使用清晰编号，例如 `1. 给我看的超短版`、`2. 一、今日一句话结论`。
- 当前页面不需要 JavaScript 高亮，但链接 hover 要有背景或左边线变化。

## HTML 生成要求

- HTML 内容必须与 Markdown 报告一致，不要减少关键信息。
- 可以为可读性重组视觉层级，但不得改变原判断含义。
- 文件名与 Markdown 同名，仅扩展名改为 `.html`。
- 页面底部注明“学习用途，不构成投资建议”。
- 如果某些数据缺失，HTML 中也必须保留“未获取 / 数据缺失 / 待确认”标记。
- 必须给每个 h2 章节生成稳定 `id`，目录中的 `<a href="#...">` 指向该 `id`。
- HTML 页面必须能在本地离线打开，不得引用外部 CSS、JS、字体或图片资源。
